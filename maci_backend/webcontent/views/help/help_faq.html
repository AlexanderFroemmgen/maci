Q: The ami does not show as a worker:

A: Did you create the ami in the correct region (default: EU Frankfurt)?

Q: Why are my worker hosts automatically shutting down?

A: It's not a bug, it's feature. Idle worker automatically shut down to safe you money.

Q: But the worker shouldn't be idle, there are open experiments.

A: Do the experiments require unmet capabilities? Is the port and address of the backend server open and reachable as set in the appsettings?

Q: Should I run the worker as root / with sudo?

A: This depends on the worker. As an example, Mininet and Mininet-Wifi require root permissions. Thus, you also have to run the worker as root.
